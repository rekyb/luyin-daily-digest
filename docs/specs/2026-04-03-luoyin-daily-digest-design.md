# Luyin Daily Digest System — Design Spec

**Date:** 2026-04-03
**Status:** Approved

\---

## Overview

An automated daily digest posted to a Slack channel every morning at 08:00 WIB. The digest surfaces Edutech, AI, and business/product news, trends, and research for the internal product and engineering team. The goal is general awareness — market signals, inspiration, and team alignment — with no human intervention required.

\---

## 1\. Architecture

```
\\\\\\\[EventBridge: cron(0 1 \\\\\\\* \\\\\\\* ? \\\\\\\*) = 08:00 WIB]
        ↓
\\\\\\\[AWS Lambda: main.py handler]
        ↓
\\\\\\\[Fetcher] — pulls RSS feeds from approved source list (sources.yaml)
        ↓
\\\\\\\[Filter \\\\\\\& Ranker] — deduplicates by URL and semantic similarity, scores by domain weight and recency
        ↓
\\\\\\\[Summarizer] — Gemini Flash generates headline + 2-3 sentence summary per item
        ↓
\\\\\\\[Insight Generator] — Gemini Flash produces 1 advisory section from Top Stories
        ↓
\\\\\\\[Formatter] — assembles Slack Block Kit message
        ↓
\\\\\\\[Publisher] — posts to Slack channel via Incoming Webhook
```

**Sections:**

1. **Header** — date and edition label
2. **Top Stories** — 5-7 items, each with headline + 2-3 sentence summary + source link
3. **Insight \& Advisory** — 1 curated section connecting the day's themes to edtech product context, with 1-2 concrete advisory points
4. **Quick Links** — 3-5 items as headline-only + link (secondary articles, research papers, tools, lower-priority news)
5. **Footer** — "This digest created by Luyin at `<time>` WIB, `<date>`"

## 3\. Content Strategy

**Domain weighting:**

|Domain|Weight|Description|
|-|-|-|
|Edutech|\~50%|Primary focus — ed innovation, learning tools, policy, research|
|AI|\~30%|LLMs, AI tools, AI in education|
|Business/Product|\~20%|Product strategy, market trends, startup news|

**Deduplication rules:**

1. URL deduplication — exact duplicate links are dropped immediately
2. Title similarity deduplication — fuzzy string matching on titles (threshold: >80% similarity); items covering the same story across sources are clustered and only the highest-quality source per cluster is kept. No embeddings required.
3. Insight layer synthesizes across Top Stories — never repeats what summaries already say

**Empty day handling:**

* If fewer than 3 items are found after deduplication, the digest is skipped for that day and no Slack message is posted (avoids low-value posts on slow news days)

## 5\. AI Model

**Model:** `gemini-2.0-flash` via Google AI Studio (free tier)

* Free tier: 1,500 requests/day, 1M tokens/minute — sufficient for daily digest
* Upgrade path: swap to `claude-sonnet-4-6` via Anthropic API with a single config change

**Tone \& Voice:**

The digest must read like it was written by a sharp, well-read human editor — not generated by AI. Prompts must explicitly enforce:

* **Banned phrases:** "it's worth noting", "delve into", "in conclusion", "importantly", "notably", "it seems", "it appears", "as we can see", "leverage", "unlock potential", "game-changer", "revolutionary", "in the realm of", "navigate the landscape"
* **No hedging:** State things directly. Drop qualifiers like "somewhat", "rather", "quite", "very"
* **No em-dash overuse:** Use sparingly, not as a default connector
* **Vary sentence length:** Mix short punchy sentences with longer ones. Avoid uniform rhythm
* **Active voice:** Write what happened, not what "has been seen to occur"
* **No buzzword soup:** Avoid stacking abstract nouns (e.g., "AI-driven learning transformation ecosystems")
* **Summaries:** Read like a sentence a journalist would write, not a product description

**Summarization:**

* Input: article content truncated to \~1,500 tokens
* Output: 2-3 sentence neutral summary written in the tone and voice rules above
* One API call per article

**Insight \& Advisory:**

* Input: all Top Story summaries (not full articles)
* Output: 2-3 paragraphs identifying cross-cutting themes + 1-2 concrete advisory points for an edtech product team, written in the tone and voice rules above
* One API call per digest run

**Fallback:**

* If Gemini API fails, digest still posts with raw headlines and links only
* Footer note: `⚠️ Summaries unavailable today`

## 6\. Infrastructure

**Runtime:** AWS Lambda (Python 3.12)

* Free tier: 1M requests/month, 400,000 GB-seconds compute
* Daily digest run uses negligible compute

**Scheduler:** AWS EventBridge

* Rule: `cron(0 1 \\\\\\\* \\\\\\\* ? \\\\\\\*)` = 08:00 WIB (UTC+7)
* Free tier covers this fully

**Secrets:** Stored as Lambda environment variables (never in code)

```
GEMINI\\\\\\\_API\\\\\\\_KEY
SLACK\\\\\\\_WEBHOOK\\\\\\\_URL
```

## 8\. Out of Scope

* Web scraping (RSS-only for now)
* Multi-channel posting
* User preferences or personalization per reader
* Analytics or read-tracking
* Storing digest history (stateless by design)
